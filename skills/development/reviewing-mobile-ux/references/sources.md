# Fuentes de referencia

Fuentes base que respaldan el criterio de esta skill. Usar como marco de evaluación, no como citas decorativas.

## Fuentes primarias

- **Nielsen Norman Group** — https://www.nngroup.com
  Usabilidad, mobile UX, touch targets y patrones de navegación.
- **web.dev (Google Chrome team)**
  - Responsive Design: https://web.dev/learn/design
  - Accessibility: https://web.dev/learn/accessibility
  - Core Web Vitals / Performance: https://web.dev/performance

## Fuentes de respaldo

- **Lyssna** — https://www.lyssna.com/blog

## Conceptos técnicos de referencia

- **Multi-input simultáneo** (web.dev — Interaction: https://web.dev/learn/design/interaction): diseñar para mouse, teclado y touch al mismo tiempo, no solo touch. Relevante cuando el mismo usuario alterna entre mobile y desktop.
- **ARIA labels** (web.dev — Accessibility: https://web.dev/learn/accessibility): los elementos interactivos deben tener descriptores accesibles para screen readers — atributos `aria-label` en íconos, botones y controles sin texto visible.
